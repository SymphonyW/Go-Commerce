# Go电商微服务项目面试高频考点

## 1. 微服务架构设计

### Q：为什么选择微服务架构？
A：微服务架构具有以下优势：
- 单一职责：每个微服务只负责一个业务领域
- 独立部署：各服务可独立开发、测试和部署
- 技术异构：不同服务可使用不同技术栈
- 弹性伸缩：可根据业务需求独立扩容

### Q：微服务拆分的原则是什么？
A：遵循以下拆分原则：
- 按业务边界拆分（如用户、商品、订单）
- 单一职责原则
- 松耦合高内聚
- 避免分布式事务

## 2. 服务通信

### Q：gRPC与RESTful API的区别？
A：
| 维度         | gRPC          | RESTful API   |
|--------------|---------------|---------------|
| 传输协议     | HTTP/2        | HTTP/1.1      |
| 数据格式     | Protobuf      | JSON          |
| 性能         | 更高          | 一般          |
| 接口定义     | 强类型        | 弱类型        |
| 流式通信     | 支持          | 不支持        |

### Q：如何保证微服务调用的可靠性？
A：
- 超时与重试机制
- 熔断器模式（Hystrix）
- 服务降级策略
- 幂等性设计

## 3. 数据管理

### Q：微服务中的数据库设计原则？
A：
- 每个微服务拥有独立数据库
- 避免跨服务数据库连接
- 使用事件驱动实现数据同步
- 最终一致性代替强一致性

### Q：Redis在购物车中的应用？
A：
- 使用Hash存储购物车商品
- 设置过期时间自动清理
- 批量操作优化性能
- 缓存击穿与雪崩解决方案

## 4. 分布式系统

### Q：分布式事务的解决方案？
A：
- 两阶段提交（2PC）
- 补偿事务（TCC）
- 本地消息表
- 最终一致性

### Q：如何实现服务发现？
A：
- 使用etcd作为服务注册中心
- 微服务启动时自动注册
- API网关动态获取服务地址
- 健康检查与故障转移

## 5. 容器化部署

### Q：Docker容器化的优势？
A：
- 环境一致性
- 快速部署
- 资源隔离
- 弹性伸缩

### Q：Docker Compose的作用？
A：
- 定义多容器应用
- 配置服务依赖
- 一键启动所有服务
- 环境变量管理
